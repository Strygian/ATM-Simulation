  ✓ C:\atm-simulation\docs\

STEP 2: COMPILE THE PROJECT
============================

Option A: Compile all files at once (RECOMMENDED)
-------------------------------------------------
Run from: C:\atm-simulation\

Command:
  javac -cp "lib/*" -d bin src\main\java\**\*.java

This will:
  - Use all JAR files in lib/ folder (including sqlite-jdbc)
  - Output compiled .class files to bin/ folder
  - Compile all Java files recursively

Option B: Compile with source path (alternative)
------------------------------------------------
Command:
  javac -cp "lib/*" -sourcepath "src\main\java" -d bin src\main\java\AppMain.java

Option C: Compile using explicit package structure
--------------------------------------------------
Command:
  javac -cp "lib/*" -d bin `
    src\main\java\model\*.java `
    src\main\java\dao\*.java `
    src\main\java\service\*.java `
    src\main\java\ui\*.java `
    src\main\java\util\*.java `
    src\main\java\AppMain.java

Expected Output:
  No errors should appear
  bin/ folder will be created with all .class files
  Structure: bin/model, bin/dao, bin/service, bin/ui, bin/util, bin/AppMain.class

TROUBLESHOOTING
===============

Problem 1: "Cannot find symbol: class User"
  Solution: Ensure compile command includes all packages
  Check: All model/*.java files are included in compile command

Problem 2: "org.sqlite.JDBC not found"
  Solution: Verify SQLite JDBC JAR is in lib/ folder
  Check: C:\atm-simulation\lib\sqlite-jdbc-3.44.0.0.jar exists
  Fix: Download from: https://github.com/xerial/sqlite-jdbc/releases

Problem 3: "Database locked"
  Solution: Close and restart application
  Fix: Delete atm_database.db file
  Note: Application will recreate it on next run

Problem 4: Compilation fails with "javac: command not found"
  Solution: Java isn't in PATH
  Fix: Install JDK from https://www.oracle.com/java/technologies/downloads/
  Verify: Run "java -version" in command prompt

Problem 5: "Cannot read db.properties"
  Solution: File isn't in classpath
  Check: src/resources/db.properties exists
  Note: Compilation copies this file when using IDEs

Problem 6: "Package not found: util"
  Solution: Check package names in Java files
  Verify: First line of each file should be "package packagename;"

RUNNING FROM DIFFERENT LOCATIONS
=================================

If database location needs to change:
  1. Edit: C:\atm-simulation\src\resources\db.properties
  2. Change: db.jdbc.url=jdbc:sqlite:/path/to/database/atm_database.db
  3. Recompile
  4. Run again

Running from IDE (IntelliJ IDEA, Eclipse, VS Code):
  1. Open project folder
  2. Configure to use Java 8+ SDK
  3. Mark src/main/java as source root
  4. Mark src/resources as resources folder
  5. Add lib/*.jar to project classpath
  6. Right-click AppMain.java → Run

CLEAN BUILD
===========

To do a clean rebuild:

PowerShell:
  rm -r bin; javac -cp "lib/*" -d bin src\main\java\**\*.java

Command Prompt:
  rmdir /s /q bin
  javac -cp "lib/*" -d bin src\main\java\**\*.java

This removes all compiled files and recompiles from scratch.
