HOW TO RUN THE APPLICATION
===========================

METHOD 1: Using the Batch Script (Windows)
-------------------------------------------
1. Open Command Prompt
2. Navigate to: C:\atm-simulation
3. Run: run.bat

METHOD 2: Using the Shell Script (Linux/Mac)
---------------------------------------------
1. Open Terminal
2. Navigate to: /path/to/atm-simulation
3. Run: bash run.sh

METHOD 3: Manual Compilation & Execution (Windows)
--------------------------------------------------
1. Open Command Prompt
2. Navigate to: C:\atm-simulation
3. Compile:
   javac -cp "lib\*" -d bin src\main\java\model\*.java ^
   src\main\java\dao\*.java src\main\java\service\*.java ^
   src\main\java\ui\*.java src\main\java\util\*.java src\main\java\AppMain.java

4. Run:
   java -cp "bin;lib\*" AppMain

DATABASE STRUCTURE
==================

Three main tables created:

USERS Table:
  user_id (Primary Key, Auto-increment)
  name (Text)
  email (Text)
  created_at (DateTime)

ACCOUNTS Table:
  account_id (Primary Key, Auto-increment)
  user_id (Foreign Key → users)
  pin_hash (Text) - Currently plain text, hash recommended for production
  balance (Real) - Dollar amount
  status (Text) - ACTIVE or LOCKED
  created_at (DateTime)

TRANSACTIONS Table:
  txn_id (Primary Key, Auto-increment)
  account_id (Foreign Key → accounts)
  type (Text) - DEPOSIT, WITHDRAWAL, BALANCE_CHECK
  amount (Real) - Transaction amount
  timestamp (DateTime)
  description (Text)
  balance_after (Real) - Balance snapshot after transaction

TROUBLESHOOTING
===============

Issue: "Cannot find or load main class AppMain"
Solution: Ensure bin/ folder exists and recompile

Issue: "Database locked"
Solution: Delete atm_database.db and restart application

Issue: "SQLite JDBC driver not found"
Solution: Verify sqlite-jdbc jar exists in lib/ folder

Issue: "No suitable driver"
Solution: Add proper classpath: java -cp "bin;lib\*" AppMain
